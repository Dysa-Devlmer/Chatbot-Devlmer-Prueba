  • Lógica interna
  • Credenciales y API keys

FLUJO COMPLETO: USUARIO ESCRIBE MENSAJE EN WHATSAPP
================================================================================

1. USUARIO (Frontend)
   ├─ Escribe en WhatsApp: "Hola, ¿cómo estás?"
   └─ VE: Chat interface bonita

2. WHATSAPP API (Internet)
   └─ Envía a servidor en: https://chatbot.zgamersa.com/api/whatsapp/webhook

3. SERVIDOR (Backend)
   ├─ webhook-auth middleware
   │  ├─ Valida HMAC signature
   │  ├─ Rate limit check
   │  └─ Extrae IP
   │
   ├─ WhatsAppService
   │  ├─ Parse payload
   │  ├─ Get user from BD
   │  └─ Create conversation
   │
   ├─ MessageProcessorService
   │  ├─ Extrae contenido
   │  ├─ Obtiene contexto (últimos 10 mensajes)
   │  ├─ Prepara prompt
   │  └─ Formatea para IA
   │
   ├─ PerplexityService
   │  ├─ Intenta Perplexity (primario)
   │  │  └─ 2-3 segundos
   │  └─ Si falla → Ollama (fallback)
   │     └─ 5-7 segundos
   │
   ├─ MessageRepository
   │  ├─ Guarda mensaje entrante
   │  ├─ Guarda respuesta de bot
   │  └─ Check duplicados
   │
   └─ WhatsAppRepository
      └─ Envía respuesta

4. USUARIO RECIBE (Frontend)
   ├─ Llega mensaje en WhatsApp
   │  └─ "¡Hola! Estoy aquí para ayudarte..."
   └─ VE: Respuesta automática en su chat

5. ADMIN VE (Frontend)
   ├─ Aparece en /inbox
   ├─ Se actualiza en tiempo real
   ├─ Aparece en /analytics
   ├─ Se guarda en historial
   └─ VE: Conversación en dashboard

¿QUIÉN HACE QUÉ?
================================================================================

USUARIO (Cliente):
  • Escribe mensajes por WhatsApp
  • Ve chat bonito en navegador
  • Accede a admin panel
  • Lee métricas
  • NO SABE: Cómo funciona internamente

FRONTEND (React):
  • Renderiza interfaz
  • Solicita datos al backend
  • Muestra respuestas
  • Actualiza en tiempo real
  • NO ACCEDE: Base de datos directamente

BACKEND (Node.js):
  • Procesa solicitudes
  • Valida seguridad
  • Accede base de datos
  • Genera respuestas con IA
  • Guarda datos
  • NO MUESTRA: Detalles técnicos al usuario

BASE DE DATOS (SQLite):
  • Almacena usuarios
  • Almacena conversaciones
  • Almacena mensajes
  • Almacena logs
  • NO DEVUELVE: Datos sin validar

IA (Perplexity/Ollama):
  • Genera respuestas
  • Entiende contexto
  • Aprende de feedback
  • NO DECIDE: Política del chatbot

BENEFICIOS DE ESTA ARQUITECTURA
================================================================================

Para el USUARIO:
  ✓ Interfaz limpia y simple
  ✓ No ve complejidad técnica
  ✓ Experiencia fluida
  ✓ Chat rápido y responsivo

Para el SISTEMA:
  ✓ Seguridad robusta en backend
  ✓ Frontend desacoplado de lógica
  ✓ Fácil de mantener y actualizar
  ✓ Escalable (frontend y backend separados)

Para la EMPRESA:
  ✓ Flexible: Cambiar backend sin afectar UI
  ✓ Seguro: Credenciales nunca en frontend
  ✓ Auditable: Todo logged en backend
  ✓ Profesional: Separación clara de responsabilidades

EJEMPLO REAL: CAMBIAR MODELO IA
================================================================================

Si quisieras cambiar de Perplexity a otro modelo:

FRONTEND (Usuario):
  NO SABE QUE SUCEDIÓ
  ✓ Sigue viendo el mismo chat
  ✓ Sigue recibiendo respuestas
  ✓ Experiencia idéntica

BACKEND (Sistema):
  ✓ Cambio en PerplexityService
  ✓ Actualiza llamada a API
  ✓ Ajusta fallback
  ✓ Listo - transparente para usuario

RESULTADO:
  ✓ Usuario nunca se entera
  ✓ Sistema adaptable
  ✓ Upgrade sin downtime

CONCLUSIÓN
────────────────────────────────────────────────────────────────────────────

PITHY Chatbot tiene una arquitectura profesional:

FRONTEND (React):
  Responsable de la EXPERIENCIA del usuario
  Usuarios ven: Interface bonita y funcional

BACKEND (Node.js):
  Responsable de la LÓGICA y SEGURIDAD
  Usuarios NO ven: Complejidad interna

Esta separación es la MEJOR PRÁCTICA en desarrollo de software.
Permite que cada equipo sea especialista en su área:
  • Diseñadores → Frontend (UX/UI)
  • Backend engineers → Backend (Lógica/Seguridad)
  • DevOps → Infraestructura

RESULTADO: Sistema robusto, seguro y escalable ✅
